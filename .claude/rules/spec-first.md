# 機能追加時のワークフロー：仕様書ファースト

機能追加・変更を行う際は、必ず以下の順序で進める：

1. **仕様書の更新** — `docs/spec-*.md` の該当ファイルを先に修正・追記する
2. **ユーザーへの確認** — 更新した仕様書の内容を提示し、合意を得る
3. **実装** — 確定した仕様書に沿ってコードを書く

## 対象

- 新機能の追加
- 既存機能の仕様変更
- APIエンドポイントの追加・変更
- DBスキーマの変更

## 仕様書が存在しない場合

新機能に対応する `spec-*.md` がなければ、新規作成してから実装に進む。
ファイル名は `spec-<機能名>.md` とする（例: `spec-payment.md`）。

## ユーザーへの提示方法

仕様書を更新したら、変更差分（追加・修正箇所）を簡潔にまとめてユーザーに提示する。
「この仕様で進めてよいですか？」と確認を取ってから実装を始める。
